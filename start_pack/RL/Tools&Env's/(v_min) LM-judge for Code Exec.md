Из-за чего была добавлена поддержка вызова LM-judge для оценки вызова интепретатора:

* помимо детерминированного чекера используется LM-judge
* на вход подаются запрос юзера + вызов интерпретатора + GT
* скор 0-5 → нормируется в 0-1
* оценка LM-judge суммируется с оценкой determ checker – со сдвигом — чтобы предотвратить рост FPR
