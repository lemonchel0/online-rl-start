# (v_min) Code Usage Taxonomy

Посмотрел на использование интепретатора в бейзлановом ране 

`debug-runs/debug_math_code_with_code_executor_baseline_run_9b_reasoning`

на валидационных генерациях, где был наибольший `AIME (pass@16)`, использовал `Qwen3-235B-A22B-Instruct-2507-FP8`

Всего \~ `6k` сэмплов было использовано для анализа


Использовал 4 составляющие:

* `Necessity Appropriateness` — насколько в принципе обосновано использование 
* `Mathematical Grounding` — как концептуально выглядит использование с точки зрения математики
* `Epistemic Role` – использование с точки зрения верификации / достоверности
* `API Correctness` – корректность с точки зрения написания кода
