<a id="мл4-w2"></a>

**МЛ-4: Reward matrix analysis**
- Задача: Получить OS reward matrix от Димы (5x7) + determ checks + B2C checkers. Провести анализ:
  - Какие rewards стабильно работают (WORK во всех доменах)?
  - Какие rewards domain-specific (WORK в 1-2 доменах)?
  - Какие rewards не работают вообще?
  - Рекомендация: reward stack для RL v0 (W3+).
- DoD: **Reward analysis report. Рекомендованный reward stack для RL v0.**

**Критерии для reward stack RL v0:**
- Safety reward: precision >= 99% (жёсткий порог).
- Math/Code rewards: F1 >= 0.80 — иначе determ-only.
- RU rewards: F1 >= 0.70 — иначе LoRA обучать (Поля W3+).
- General rewards: F1 >= 0.75.

## Конфигурация eval sweep

**Для воспроизводимости фиксируем:**

```yaml
eval_config:
  models:
    - name: 10b_toolmind
      path: /path/to/10b_toolmind
      backend: vllm
    - name: qwen3.5-2b
      path: /path/or/hf/id
      backend: vllm
    - name: qwen3.5-4b
      path: /path/or/hf/id
      backend: vllm
    - name: glm-5
      path: /path/or/hf/id
      backend: vllm
    - name: deepseek-v3.2
      path: /path/or/hf/id
      backend: vllm

  benchmarks:
    math: [math500, aime, gsm8k]
    code: [lcb_v6, swe_bench]
    if: [ifeval_strict, ifeval_loose, vikacheck]
    tools: [bfcl_v4]
    arena: [arena_hard_lc]
    ru: [mera, rusifikator_test, ty_vy_test, smena_test]
    safety: [internal_safety, refusal_rate]
    obm: [obm_format_50]

  generation:
    temperature: 0.0
    max_tokens: 4096
    n_samples: 1  # pass@1 для большинства; AIME может потребовать pass@4

  output:
    format: json + markdown
    path: s3://reward-test/eval-results/baseline-w2/
```
